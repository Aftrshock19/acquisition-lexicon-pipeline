Absolutely. Here is a **large drop in template bank spec** for the long sentence repair system.

I am going to structure it so you can use it directly in code or convert it to JSON, YAML, CSV, or Python dicts.

The design goal is this:

1. keep the **exact target surface form**
2. keep the sentence **natural**
3. keep it **short**
4. keep it **self contained**
5. make it easy to generate a **clean English translation**

Everything below is built for the repair stage where the original sentence is too long, too dense, too legal, too weird, or too broken.

---

# 1. Core design contract

Every repair template should aim for:

1. **Preferred length**: 4 to 10 words
2. **Hard maximum**: 12 words
3. **Clause count**: one clause preferred
4. **Target visibility**: target form must be central
5. **Support vocabulary**: high frequency when possible
6. **Translation ease**: should translate cleanly into short English

Each template should be represented with metadata like this:

```python
{
    "template_id": "v_pres_01",
    "pos_family": "verb",
    "mood": "indicative",
    "tense": "present",
    "person": "1s",
    "pattern": "Yo {target} {obj}.",
    "slots": ["target", "obj"],
    "slot_constraints": {
        "obj": ["concrete_noun", "abstract_noun", "pronoun_optional"]
    },
    "preferred_word_count_min": 3,
    "preferred_word_count_max": 6,
    "register": "neutral",
    "difficulty": "easy",
    "translation_pattern": "I {target_en} {obj_en}.",
    "notes": "Very safe for many transitive present forms"
}
```

You do not need every template to be this detailed at first, but this is the ideal structure.

# 3. Global selection rules

Before templates, here is the rule order the repair system should use.

## Rule 1

Prefer templates with the fewest slots.

A 2 slot template is safer than a 4 slot template.

## Rule 2

Prefer concrete nouns over abstract nouns unless the target strongly prefers abstract context.

## Rule 3

Prefer declarative sentences over conditionals, relatives, and subordinate structures.

## Rule 4

Only use subordinate triggers like `que`, `si`, `aunque`, `cuando`, `hasta que` when the target form really needs them.

## Rule 5

If the target is a fragile function word, use hand built templates only.

## Rule 6

If the target is rare or semantically specific, choose a semantically compatible noun set rather than generic filler.

## Rule 7

If a generated result exceeds 12 words, back off to a smaller template, not a trim.

```python
    {"template_id": "v_subj_pres_05", "pattern": "Es mejor que {target}.", "translation": "It is better that {target_en_subj}."},
    {"template_id": "v_subj_pres_06", "pattern": "Quiero que {target} aquí.", "translation": "I want {target_en_subj} here."},
    {"template_id": "v_subj_pres_07", "pattern": "Espero que {target} bien.", "translation": "I hope {target_en_subj} well."},
    {"template_id": "v_subj_pres_08", "pattern": "Ojalá {target} pronto.", "translation": "Hopefully {target_en_subj} soon."},
]
```

Examples:
`hagan`, `salgan`, `esté`, `sepa`, `vaya`, `puedan`, `quiera`.

### Better English mapping note

For subjunctive, you often do **not** want literal “that he do”.
You want something like:

* `Espero que hagan lo correcto.` → `I hope they do the right thing.`
* `Es posible que salgan hoy.` → `They may leave today.`

So your template metadata should allow non literal English glossing.

# 16. POS family routing logic

You need a router before template selection.

```python
def choose_template_family(row):
    if row.pos_family == "verb":
        return choose_verb_template_family(row)
    if row.pos_family == "noun":
        return choose_noun_template_family(row)
    if row.pos_family == "adjective":
        return choose_adjective_template_family(row)
    if row.pos_family == "adverb":
        return ADVERB_TEMPLATES
    if row.pos_family == "pronoun":
        return choose_pronoun_template_family(row)
    if row.pos_family == "article":
        return ARTICLE_DEFINITE or ARTICLE_INDEFINITE
    if row.pos_family == "determiner":
        return DETERMINER_TEMPLATES
    if row.pos_family == "preposition":
        return choose_preposition_bank(row.lemma)
    if row.pos_family == "conjunction":
        return choose_conjunction_bank(row.lemma)
    if row.pos_family == "numeral":
        return NUMERAL_TEMPLATES
    return None
```

For verbs:

```python
def choose_verb_template_family(row):
    mood = row.get("mood")
    tense = row.get("tense")
    person = row.get("person")

    if mood == "indicative" and tense == "present":
        return VERB_INDICATIVE_PRESENT_1S if person == "1s" else ...
    if mood == "indicative" and tense == "preterite":
        return VERB_PRETERITE
    if mood == "indicative" and tense == "imperfect":
        return VERB_IMPERFECT
    if mood == "indicative" and tense == "future":
        return VERB_FUTURE
    if mood == "subjunctive" and tense == "present":
        return VERB_SUBJ_PRESENT
    if mood == "subjunctive" and tense == "imperfect":
        return VERB_SUBJ_IMPERFECT
    if mood == "subjunctive" and tense == "present_perfect":
        return VERB_SUBJ_PERFECT
    if mood == "infinitive":
        return VERB_INFINITIVE
    if mood == "gerund":
        return VERB_GERUND
    if mood == "participle":
        return VERB_PARTICIPLE
    return fallback_verb_templates
```

# 17. Slot filling rules

The template bank only works if slot filling is careful.

## Safe slot filling order

1. use fixed lexical material already built into the template if possible
2. use high frequency abstract objects for abstract verbs
3. use concrete objects for transitive action verbs
4. use place and time adverbs only if needed
5. never add more than one adjunct unless the template was designed for it

## Example slot selection

```python
def choose_object_for_verb(lemma, semantic_class):
    if semantic_class == "communication":
        return random.choice(["la verdad", "eso", "la noticia"])
    if semantic_class == "physical_action":
        return random.choice(["la puerta", "la caja", "la carta"])
    if semantic_class == "mental":
        return random.choice(["la idea", "el problema", "eso"])
    return random.choice(["eso", "el plan", "la verdad"])
```

This becomes much better if you later add verb semantic classes.

# 19. Validation rules for template output

4. punctuation is simple
5. no unresolved pronoun unless acceptable
6. sentence begins naturally
7. no article gender mismatch
8. no number mismatch
9. no bizarre semantic pairing

1. short
2. faithful
3. tense matches
4. person matches
5. not more specific than Spanish
6. not literal in a bad way
